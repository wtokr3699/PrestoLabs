## 🔴 Critical / High

### F1. 본인 거래·본인 리뷰 조작 체인 ⬜
한 사용자가 혼자서 "완료 프로젝트 + 자기 5점 리뷰"를 조작해 평점을 띄울 수 있음.
- **F1a** `applications/[id]/applications` POST: `project.clientId === uid` 차단 없음 — `src/app/api/projects/[id]/applications/route.ts:52`
- **F1b** `reviews` POST: `revieweeId === uid` 자기리뷰 차단 없음 — `src/app/api/reviews/route.ts:39`
- **F1c** `auth/me` PATCH: 활성 프로젝트/지원 보유 중 역할 전환 무제한 — `src/app/api/auth/me/route.ts:37`

### F2. 지원 수락 상태 가드 전무 → 중복 매칭·이중 계약 ⬜
`accept/route.ts`가 `application.status`/`project.status`를 확인하지 않음. 이미 matched된 프로젝트에 재수락 시 계약 중복 생성. — `src/app/api/applications/[id]/accept/route.ts:12`

### F3. 상태 변경 API가 매칭/완료 단계를 우회 허용 ⬜
상태머신이 `in_review→matched`, `submitted→completed` 등을 허용해, 계약/결제 없이 곧장 전환 가능. — `src/app/api/projects/[id]/status/route.ts:21`

### F4. 예산 변경 잠금 우회 ⬜
`apply-budget` 라우트에 "지원자 있으면 변경 불가" 가드 없음 + `budgetMin<=budgetMax` 검증 없음. — `src/app/api/projects/[id]/apply-budget/route.ts:6`

### F5. 계약 서명 상태 가드 없음 → 완료 계약이 active로 회귀 ⬜
`sign/route.ts`가 `status` 확인 없이 양측 서명 시 active로 전환. completed 계약 회귀 가능. — `src/app/api/contracts/[id]/sign/route.ts:15`

### F6. 리뷰 평점 타입 미검증 → avgRating NaN 오염 ⬜
`rating`을 숫자 강제하지 않아 문자열/NaN이 평균에 누적. — `src/app/api/reviews/route.ts:14`

### F7. 크론 인증 fail-open ⬜
`CRON_SECRET` 미설정 시 `"Bearer undefined"`로 통과. — `src/app/api/cron/deadline/route.ts:14`

### F8. 시드 라우트 fail-open + 실데이터 덮어쓰기/삭제 ⬜
`SEED_SECRET` 미설정 + `NODE_ENV!=="production"`이면 무인증 허용. `set()` merge 없음. — `src/lib/seedAccess.ts:7`

## 🟠 Medium

### F9. 샌드박스 결제 우회가 NODE_ENV에만 의존 ⬜
`paymentKey`가 `sandbox_pk_`면 Toss 검증 생략. 명시 플래그로 격리. — `src/app/api/payments/confirm/route.ts:51`

### F10. 납품 요청 없이 에스크로 정산 가능 ⬜
`approve`가 `deliveryRequestedAt` 확인 없이 정산. — `src/app/api/contracts/[id]/approve/route.ts:16`

### F11. 결제 금액 검증 누락 + 중복 pending 경쟁 ⬜
`initiate`가 `agreedBudget` 양의 정수 검증 없음, existing-check/add 비원자. — `src/app/api/payments/initiate/route.ts:21`

### F12. 지원 카운터/중복 비원자 ⬜
`applicationCount + 1`, 사전읽기 중복검사. `FieldValue.increment` + 결정적 ID 권장. — `src/app/api/projects/[id]/applications/route.ts:69`

### F13. 헬스 엔드포인트 정보 노출 ⬜
무인증으로 env 존재 여부·`String(err)`·DB 읽기 노출. — `src/app/api/health/route.ts:1`

### F14. profileComplete 의도치 않은 false화 / 역할 전환 필드 미재검증 ⬜
PATCH마다 재계산. — `src/app/api/auth/me/route.ts:51`

### F15. 프로젝트 삭제 비연쇄 + 안내문 불일치 ⬜
soft delete만, 편집 페이지는 "지원서도 삭제" 안내. `applications/my`가 삭제 프로젝트 노출. — `src/app/api/projects/[id]/route.ts:72`

### F16. 확장성: 전 컬렉션 로딩 ⬜
프로젝트/전문가/admin stats/알림/북마크가 limit·인덱스 없이 전체 로딩. — 여러 파일

### F17. notifications read-all 500건 초과 시 실패 ⬜
단일 batch가 500 write 한도 초과 시 throw. — `src/app/api/notifications/read-all/route.ts:9`

### F18. payments GET이 pgPaymentKey 노출 ⬜
응답에서 PG 키 제거. — `src/app/api/payments/[id]/route.ts:1`
