# PrestoLabs 통합 서비스 기획서

## 1. 문서 목적
이 문서는 팀원들이 작성한 `WorkBridge`, `BRIDGE`, `PrestoLabs`, `뚝딱연구소` 기획안을 하나의 서비스 전략으로 통합한 최종 초안이다. 이후 웹사이트 구축, 발표 자료 제작, MVP 구현의 기준 문서로 사용한다.

## 2. 최종 서비스 정의
### 서비스명
PrestoLabs

### 브랜드 한 줄 정의
아이디어만 입력하면 AI가 요구사항을 정리하고, 가장 적합한 실행자를 매칭해 48시간 안에 작동하는 결과물로 연결하는 AI 기반 MVP 제작 플랫폼.

### 슬로건
48시간 안에, 아이디어를 작동하는 결과물로.

## 3. 통합 방향
팀 문서들의 의견을 다음 4가지 축으로 통합한다.

1. `PrestoLabs`의 브랜드 방향을 채택한다.
브랜드 톤은 빠르고 친근하지만, 실제 서비스는 신뢰와 실행력을 전면에 둔다.

2. `WorkBridge`의 거래 신뢰 구조를 채택한다.
AI 요구사항 정리, 계약서 생성, 에스크로 결제, 검수 체크리스트, 리뷰 구조를 기본 골격으로 삼는다.

3. `뚝딱연구소`의 48시간 MVP 실행 모델을 핵심 차별점으로 채택한다.
서비스는 "모든 것을 대신 개발해 주는 일반 외주"가 아니라 "짧은 시간 안에 핵심 기능이 작동하는 MVP"를 만드는 플랫폼으로 정의한다.

4. `BRIDGE`의 UX 관점을 채택한다.
의뢰인은 전문가를 검색하는 대신, AI와 대화하며 아이디어를 구조화하고 추천받는 흐름을 기본 경험으로 설계한다.

## 4. 문제 정의
### 의뢰인의 문제
- 아이디어는 있지만 어디서부터 시작해야 할지 모른다.
- 개발자에게 무엇을 어떻게 설명해야 하는지 모른다.
- 견적이 적정한지 판단하기 어렵다.
- 외주 진행 중 범위가 바뀌거나 추가 비용이 붙을까 불안하다.
- 결과물이 언제 나오고 어떤 수준일지 예측하기 어렵다.

### 제작자의 문제
- AI를 활용한 빠른 제작 역량을 가격과 신뢰로 인정받기 어렵다.
- 작은 프로젝트는 요구사항 정리 비용이 커서 수익성이 낮다.
- 기존 플랫폼은 저가 경쟁이나 소모적인 입찰 구조가 강하다.

### 시장의 공백
기존 플랫폼은 전문가 탐색 중심이다. 하지만 실제로는 `요구사항 정리`, `작업 범위 합의`, `검수 기준`, `결제 안전장치`까지 포함된 실행 프로세스가 부족하다. PrestoLabs는 이 빈칸을 채운다.

## 5. 핵심 가치 제안
### 1) 아이디어만 있어도 시작 가능
사용자는 기획서 없이 자연어로 문제와 원하는 결과를 설명하면 된다. AI가 이를 요구사항 문서로 정리한다.

### 2) AI가 기획과 매칭을 동시에 돕는다
AI는 아이디어를 기능 단위로 정리하고, 예상 난이도와 적합한 실행자 유형을 분석해 추천한다.

### 3) 48시간 실행 모델
단순하고 명확한 MVP, 자동화, 웹페이지, 내부 도구는 48시간 안에 완료 가능한 범위로 구조화한다. 더 큰 프로젝트는 2뚝딱, 3뚝딱처럼 단위를 확장한다.

### 4) 안전한 검수와 정산
요구사항 문서, 체크리스트, 에스크로 결제, 단계별 상태 표시를 통해 의뢰인은 불안을 줄이고 제작자는 정산 리스크를 줄인다.

## 6. 서비스 구조
PrestoLabs는 두 개의 트랙으로 운영한다.

### 트랙 A. 빠른 패키지형
자주 요청되는 기능을 표준화한 상품형 서비스다.

예시:
- 랜딩페이지 제작
- 예약/문의 관리 시스템
- 구글시트 자동화
- 내부 업무 자동화 봇
- 간단한 관리자 페이지

특징:
- 가격이 명확하다.
- 작업 범위가 분명하다.
- 빠른 의사결정과 즉시 구매 전환에 유리하다.

### 트랙 B. 커스텀 매칭형
사용자의 아이디어를 기반으로 AI가 요구사항을 정리하고, 적합한 전문가 또는 스쿼드를 추천하는 맞춤형 서비스다.

특징:
- 커스텀 요구에 대응 가능하다.
- 매칭 추천, 포트폴리오 비교, 검수 흐름이 중요하다.
- 향후 PrestoLabs의 핵심 확장 축이다.

## 7. 핵심 사용자
### 1차 타깃
- 예비 창업자
- 소상공인
- 1인 사업자
- 대학생 팀
- 업무 자동화가 필요한 실무자

### 2차 타깃
- 소규모 스타트업
- 내부 도구가 필요한 중소기업
- 고객 응대 자동화가 필요한 운영팀

### 공급자 타깃
- AI 활용 개발자
- 웹앱 제작자
- 노코드/로우코드 자동화 전문가
- 챗봇 제작자
- 기획 및 MVP 설계가 가능한 디자이너/프리랜서

## 8. 핵심 기능
### 1) AI 기획 번역기
사용자의 자연어 아이디어를 질문형 대화로 정리해 다음 결과를 만든다.
- 문제 정의
- 주요 사용자
- 핵심 기능 목록
- 제외 범위
- 추천 결과물 형태
- 예상 작업 난이도
- 예상 기간

### 2) AI 전문가 매칭
AI가 프로젝트 요구사항과 실행자 프로필을 비교해 추천한다.

매칭 기준:
- 전문 분야
- 유사 프로젝트 경험
- 포트폴리오 적합도
- 작업 가능 시간
- 48시간 수행 가능 여부
- 평점과 완료율

### 3) 프로젝트 등록과 비교
의뢰인은 AI가 정리한 기획서를 바탕으로 프로젝트를 등록하고, 추천 실행자 또는 제안 카드를 비교할 수 있다.

### 4) 계약 및 결제 안전장치
- 전자 계약서 생성
- 에스크로 결제
- 상태별 진행 관리
- 결과물 제출 및 승인

### 5) 검수 체크리스트
AI가 요구사항서 기반으로 체크리스트를 만들어, 결과물 검수를 감정이 아니라 합의된 항목 기준으로 진행하게 한다.

### 6) 진행 상황 대시보드
- 남은 시간
- 현재 단계
- 채팅
- 파일 공유
- 중간 산출물
- 수정 요청

## 9. 사용자 여정
### 의뢰인
1. 메인 페이지에서 아이디어 입력
2. AI가 요구사항과 핵심 기능 정리
3. 예산, 원하는 결과물, 우선순위 확인
4. 추천 실행자 또는 패키지 확인
5. 선택 후 결제 및 프로젝트 시작
6. 48시간 카운트다운과 진행 현황 확인
7. 결과물 검수 후 승인 또는 수정 요청
8. 리뷰 작성

### 실행자
1. 프로필과 포트폴리오 등록
2. 기술 분야 및 작업 가능 범위 설정
3. 추천 프로젝트 확인
4. 참여 또는 제안서 제출
5. 계약 체결 후 작업 진행
6. 중간 산출물 공유
7. 최종 결과물 제출
8. 검수 완료 후 정산

## 11. 가격/정산 원칙
### MVP 단계 원칙
- 패키지형은 고정가
- 커스텀형은 AI 분석 기반 예상가 제시
- 시작 전 총액과 범위를 최대한 명확히 고지

### 수익 모델
1. 거래 수수료
2. 직영 패키지 판매
3. 유지보수 구독
4. 빠른 매칭/우선 노출 옵션
5. 향후 프리미엄 운영 관리 플랜

## 12. 운영 정책 초안
### 48시간 기준
48시간 카운트는 `요구사항 확정 + 결제 완료 + 작업 시작` 시점부터 시작한다.

### 수정 정책
- 기본 수정 1~2회 포함
- 범위 외 요청은 추가 비용 또는 추가 뚝딱으로 분리

### 환불/보완 원칙
- 작업 시작 전 전액 환불 가능
- 작업 시작 후에는 진행 단계와 체크리스트 기준으로 부분 보완 또는 부분 환불
- 감정적 불만족이 아니라 합의된 요구사항 충족 여부를 기준으로 판단

## 15. 브랜드/디자인 가이드
김태민 디자인 시안의 방향을 웹에 반영한다.

핵심 원칙:
- 차분한 뉴트럴 배경 위에 얇은 보더와 카드 레이아웃 사용
- 과장된 스타트업 스타일보다 정리된 편집형 레이아웃 사용
- 정보는 카드, 스텝, 타임라인, 프리뷰 패널로 나누어 전달
- 신뢰감 있는 여백과 섬세한 계층 구조 유지
- 강조 색은 제한적으로 사용하고, 전체 톤은 미니멀하게 유지

- 브랜드명은 `PrestoLabs`
